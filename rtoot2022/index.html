What is Mastodon? ü¶£

mastodon is (not) an open source twitter clone


Mastodon is a software to host microblogging social networking sites
tweets (280 char) ‚Üí toots (500 char)
retweet ‚Üí boost
like ‚Üí favorite
quote tweet ‚Üí üö´


But
it‚Äôs not a single application!
it‚Äôs a distributed network of servers that interoperate
servers form a large decentralized federation
no ads, no insentives for ‚Äúgoing viral‚Äù, no ‚Äúalgorithms‚Äù

centralized/federated/distributed

image from https://docs.joinmastodon.org/

centralized: Twitter, Facebook, Instagram
federated: Email, phone networks, physical mail
distributed: BitTorrent

Why does decentralization matter?


  • reduced operating costs ‚áí no need for big monetization schemes
  • servers don‚Äôt need to grow beyond their financial capacity
  • more resilient to censorship (positive/negative)
  • more control over your own data

The Fediverse

The fediverse (‚Äúfederated universe‚Äù) is the name for all websites that can communicate with each other over ActivityPub and the World Wide Web.

  • Mastodon: Fediverse alternative to Twitter
  • PeerTube: Fediverse alternative to YouTube
  • PixelFed: Fediverse alternative to Instagram
  • Lemmy: Fediverse alternative to Reddit
  • Friendica: Fediverse alternative to Facebook

But wait, there is more



  • Tumblr will join the Fediverse
  • BookWyrm (like goodreads)
  • FunkWhale (like soundcloud)


There is a Fediverse alternative for almost anything

Authentication


API calls can be made anonymously, with a client, or user token.

auth_setup()

The function guides you through the process of authentication


By default, the token is stored in a directory R has determined to make sense for you and it is called ‚Äúrtoot_token.rds‚Äù.
You do not need to worry about it if you are only using rtoot on your local machine and you are only using one token.

Consult the vignette for more advanced authentication options

vignette("auth",package = "rtoot")

understanding timelines


home timeline: all statuses of accounts you follow

local timeline: all statuses of an instance

list timeline: all statuses of a list of accounts

hashtag timeline: all statuses including a specific hashtag

public statuses: all statuses of accesible Mastodon servers

Examples

get_timeline_public(instance = "mastodon.social",local = TRUE,limit = 5)

The statuses include a list column account which holds all account data.
The acct field has the form user_name@instance

Post a toot

on some instances, automated posting is frowned upon
(some reports of being ‚Äúshadow banned‚Äù)

# post a simple status
post_toot("my first rtoot #rstats")

# post a media file with alt text
post_toot("look at this pic",media = "path/to/image",
          alt_text = "describe image", sensitive = TRUE)
  • posts can be scheduled using the parameter scheduled_at (5 minutes in future)
  • spoiler_text is used as a content warning (careful with terminology!)

User actions


perform actions on behalf of a user (your own account)

# follow 
post_user(id, action = "follow")

# unfollow 
post_user(id, action = "unfollow")

# also: (un)mute, (un)block, (un)pin, note
